第一次推送

git init 初始化本地仓库，这时候回生成一个.git目录（是隐藏文件）

git add . 把所有文件添加到暂存区，待commit，Android Studio可以设置新建文件自动add

git commit -m "备注" 将暂存区的 文件提交到主分支上

git remote add origin 链接(git@github.com:xxxxxxxxxxxxxx.git) 添加远程仓库

git push -u origin master #初始化推送

pull和push

git pull origin 拉取与当前分支有联系的远程分支，一般在多人合作的时候先拉下来解决冲突再push

git push origin 远程分支名提交代码到某个远程分支

分支相关

git branch 查看当前本地分支

git branch -a 查看所有分支，包括本地和远程

git branch -d <本地分支名如：master> 删除本地分支

git checkout -b <本地分支名如：master> 复制当前分支到另一个本地分支

解决Git中fatal: refusing to merge unrelated histories

在pull/push操作命令后面加--allow-unrelated-histories 那时候是因为在新建远程仓库时候多了个readme，不是空的，所以直接推送的时候会警告说不能合并不相关的历史
